## 2. 핵심 원칙 및 금지 규약 (Negative Constraints)

시스템 오차($\epsilon$) 폭발을 막기 위해 에이전트는 다음을 엄수한다.

* **[격리 원칙]** 지정된 샌드박스 경로 외부의 파일 시스템을 임의로 탐색하거나 수정하지 않는다.
* **[상태 독립성]** 암묵적 공유 상태(전역 변수, 인메모리 세션 등)를 금지한다. 모든 영속성 상태는 $D_p$에 명시적으로 기록한다.
* **[접근 통제]** 외부 모듈의 자체 DB는 절대 직접 조회하지 않고 지정된 API(Wrapper)를 통해서만 접근한다.
* **[멱등성 규약]** 실패한 Task를 재시도할 경우, 시스템이 오염되지 않도록 이전 상태 롤백(Revert/Clean)을 선행한 후 재실행한다.
* **[연산 한계]** 개별 로직($L_n$) 내부에서 다른 로직을 직접 호출(흐름 제어)하지 않는다. 흐름 제어는 전적으로 오케스트레이터($So$)의 역할이다.
